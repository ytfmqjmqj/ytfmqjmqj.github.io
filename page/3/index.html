48.不用加减乘除做加法

发表于 2020-06-17 更新于 2020-07-02 分类于 target-offer

写一个函数，求两个整数之和，要求在函数体内不得使用+、-、*、/四则运算符号。

思路：位运算
两个数异或：相当于每一位相加，而不考虑进位；
两个数相与，并左移一位：相当于求得进位；
将上述两步的结果相加。

Add1:
在Python内部对整数的处理分为普通整数和长整数，普通整数长度为机器位长，通常都是32位，超过这个范围的整数就自动当
长整数处理，而长整数的范围几乎完全没限制。当-1 + -6 通过while中的位运算操作之后，输出的计算结果4294967289。
经过(num1^0xffffffff)这个操作会被作为负数解释输出，运算符除了求反，还是二进制的补运算符，运算过后的二进制数字按照补码解释。就可以输出负数了。也可以使用num1 - 4294967295(0xffffffff)，也可以得到负数结果。负数是以补码形式存储的，python的取反运算结果是补码。 前面用 a & 0xFFFFFFFF 相当于无视符号位，直接转为无符号数，也就是说，这个时候解释器已经不认识补码了。后面再用一次取反，重新给强调一下，是补码。

Add2:
a & 0xFFFFFFFF的作用应该是截段a的长度。对于~(num1^0xffffffff)，举一个简单点的例子：
-15的原码是10001111，补码是11110001，-15 & 0xff = 11110001(-15的补码参与运算，对于按位与，1变成1，0变成0），而-15^0xff = 00001110（-15的补码参与运算，对于按位异或，1变成0，0变成1），再对前面的结果按位取反，从而又重新得到-15的补码11110001，相当于两次取反得到-15的原补码。

发表于 2020-06-16 更新于 2020-06-17 分类于 target-offer

如果没有小朋友，请返回-1

方法3
把n个人的编号改为0n-1，然后对删除的过程进行分析。
第一个删除的数字是(m-1)%n，记位k，则剩余的编号为(0,1,…,k-1,k+1,…,n-1)，下次开始删除时，
顺序为(k+1,…,n-1,0,1,…k-1)。
用f(n,m)表示从(0n-1)开始删除后的最终结果。
用q(n-1,m)表示从(k+1,…,n-1,0,1,…k-1)开始删除后的最终结果。
则f(n,m)=q(n-1,m)。

下面把(k+1,…,n-1,0,1,…k-1)转换为(0~n-2)的形式，即
x y
k+1对应0
k+2对于1
…
k-1对应n-2
转化函数设为p(x)=(x-k-1)%n, p(x)的逆函数为p^(x)=(x+k+1)%n。
则f(n,m)=q(n-1,m)=p^(f(n-1,m))=(f(n-1,m)+k+1)%n，又因为k=(m-1)%n。
f(n,m)=(f(n-1,m)+m)%n;
Note：注意q(n-1, m)对应的是左边列的一个数，而f(n-1, m)是右边列的一个数，两则之间可由p(x)转化。

最终的递推关系式为
f(1,m) = 0; (n=1)
f(n,m)=(f(n-1,m)+m)%n; （n>1）

42.和为S的两个数字

如果有多对数字的和等于S，输出两个数的乘积最小的。

两头开始找匹配，乘积最小必定为最先找到的。
也就是说依然是左右夹逼法！！！只需要2个指针
1.left开头，right指向结尾。
2.如果和小于sum，说明太小了，left右移寻找更大的数。
3.如果和大于sum，说明太大了，right左移寻找更小的数。
4.和相等，把left和right的数返回
Note:left->,<-right。

18,19,20,21,22。现在把问题交给你,你能不能也很快的找出所有和为S的连续正数序列? Good Luck!

输出描述:
输出所有和为S的连续正数序列。序列内按照从小至大的顺序，序列间按照开始数字从小到大的顺序

两个指针small,big。
同样有small, big, 初始值为1, 2,target为N。
序列和(small到big) > target，同样要缩小和， small向后移动一位，则序列和减小small。
否则增大和，big向后移动一位，序列和增大big。
如果相同，则保存下来。然后small向前移动一位， big向前移动一位。
如此循环直到 small >= (1 + target) /2， 因为此时big + small 肯定大于target(题目要求最少两位）
Note:small和big都只能往后移。

40.数组中只出现一次的数字

发表于 2020-06-16 分类于 target-offer

一个整型数组里除了两个数字之外，其他的数字都出现了两次。请写程序找出这两个只出现一次的数字。

首先：位运算中异或的性质：两个相同数字异或=0，一个数和0异或还是它本身。
当只有一个数出现一次时，我们把数组中所有的数，依次异或运算，最后剩下的就是落单的数，因为成对儿出现的都抵消了。
依照这个思路，我们来看两个数（我们假设是AB）出现一次的数组。我们首先还是先异或，剩下的数字肯定是A、B异或的结果，这个结果的二进制中的1，表现的是A和B的不同的位。我们就取第一个1所在的位数，假设是第3位，接着把原数组分成两组，分组标准是第3位是否为1。如此，相同的数肯定在一个组，因为相同数字所有位都相同，而不同的数，肯定不在一组。
然后把这两个组按照最开始的思路，依次异或，剩余的两个结果就是这两个只出现一次的数字。
NOTE:异或满足结合律和交换律。
